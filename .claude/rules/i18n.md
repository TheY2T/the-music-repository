# Localization / i18n (ADR 0017, `docs/features/i18n.md`)

Web UI is English + 中文 (`zh-Hans`), URL-prefix routed (`/zh/…`), gated by `platform.i18n`. Follow the
**`add-translations`** skill.

- **No hardcoded UI strings.** Every user-facing string goes through `t(locale, key)` from
  `@TheY2T/tmr-i18n`. Keys live in `@TheY2T/tmr-i18n-locales`: **`en.json` is the source of truth for keys**
  (→ `type MessageKey`); `zh-Hans.json` is Simplified Chinese (missing keys fall back to English). Add a
  key to **both** files.
- **Locale** resolves per request in `middleware.ts` → `Astro.locals.locale` (URL prefix > cookie >
  Accept-Language). `/zh/…` rewrites to the canonical page path (single page-file set).
- **Islands** take `locale: Locale` as a **plain prop** (deterministic from the URL → no hydration flash;
  React context can't cross islands) and call `t(locale, key)`. Build internal links with
  `localizedPath(locale, '/path')`; build `<title>` inline.
- **No glyphs in strings** — icons come from the `Icon` atom, not baked into i18n text (see
  `.claude/rules/design-system.md`).
- **Left as-is:** music-theory tokens, technical names, and API/DB data (catalogue titles, taxonomy).
  Backend/DB-content i18n is deferred.
